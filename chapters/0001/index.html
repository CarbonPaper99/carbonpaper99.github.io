1.1 为什么我需要一个服务器

假设你是一位小白，你有一个Minecraft世界，但它文件在你自己的电脑里。现在，你想要这个世界对外开放，与其他玩家共享此世界，怎么办？

Minecraft倒是有一个自带的功能叫局域网开放，玩家在该世界的游戏界面内点击菜单中局域网开放的按钮就可以将自己的世界开放出去。完成操作后，此世界在你电脑连接的局域网（可以是同一路由器下的有线连接网络，也可以是无线的Wifi）某一个端口开放并存在着。其他玩家所使用的设备可以检索到你的这个世界。这是个很方便的设计，因为简单而易学，网内联机多常使用这个功能。

这种简单的方法实在是傻瓜式的操作，但随着世界规模的发展，一些个人开放者发现仅仅这样做满足不了他们的需求。其一，局域网世界无法长期稳定存在，因为局域网的本体还是Java基础上运行的游戏端，跑起来效率紧追学步车。第二，局域网世界扩展功能仅局限于Mod，但要让每一个连接此世界的玩家安装上相同的Mod更麻烦。第三，规模一旦大，管理也是难事。

不要用局域网游戏玩大型多人游戏……你的电脑撑不住。

于是Mojang又煞费苦心地开发出一款同样基于Java平台的应用程序，旨在给个人的服务器社区开发者提供性能高效，管理便捷的多人游戏平台解决方案。这类软件在发布后几年时间内迅速成熟，发展出一个庞大的体系。这种软件被称为Minecraft 服务端（Minecraft Server）。用该软件搭建出来的多人游戏平台称为 Minecraft 服务器。

服务端是独立于游戏的一种应用程序，这表明它可以脱离主机游戏的限制而独立工作。这为大型多人游戏的出现提供了基础。

所以，看到这里你就能想明白了，要想搭建好多人游戏平台，走服务器道路是少不了的。这个社区除了给你带来当管理者的快感以外，也能让你在沟通中获得快乐。

如果你想搭建一个服务器，而你一窍不通，你还等什么，现在就开始，从零搭建起一个服务器，Getting Started!

1.2 原服务端的搭建过程

O1 服务端的认识及选择

一般来讲，原服务端（Original Server）是最基本的服务端。这类服务端保持着官方的出厂设置，游戏功能的调节需要玩家自己配置。这有点类似于雕刻艺术，在其中，未经雕刻、保持原貌的大理石就是一个原服务端。因此，想要从零开始一个服务器的建设，我们先从原服务端的搭建开始学起：

前面提到，服务端是一类软件的总称，这是因为多人游戏在近几年需求量大增，开发者针对不同的需求和问题开发出了许多种不同的变体服务端。在这里，我们只需要认识其中几种代表性的服务端。这其中有纯净服务端和扩展型服务端。这些服务端基本涵盖了大部分功能，以下是几个代表：（名称第二行是中文译名）

不同的服务端对应不同的需求。如果你想开一个Mod服务器，选择以上Sponge（海绵）端、KCauldron（坩埚）端等服务端即可。如果你只是想回归质朴，可以使用官服端。如果你想有一些可玩性，但是又不想整Mod服务器，需要有较好的优化等，Spigot（水龙头）与Bukkit&CraftBukkit（水桶）系列都可以满足你。

在这里为了统一，我们将采用PaperSpigot这一个以水龙头Spigot端为基础的分支服务端作为教学服务端。这个服务端基本覆盖了大部分版本，也自带优化与反作弊。需要注意的是，如果你选择部分Mod端，那么在下面的内容中你可能会发现操作比较困难，因此 建议小白先学好纯净的插件服务端的操作，再学习Mod服务端，这将会大大提升你的学习效率。

  • 教学用端 PaperSpigot https://papermc.io/legacy
  • 标准 Spigot https://getbukkit.org/download/spigot
  • 标准 CraftBukkit https://getbukkit.org/download/craftbukkit

如上是部分纯净端的下载方式，为了使链接长久能使用，选用了国外的官网地址。在进入到链接页面后，可以发现很多不同版本的服务端，选择自己的游戏版本并点击Download字样（部分页面是一个下载的图标）的按钮，就会自动开启服务端的下载，等待下载完毕即可。服务端可以自选，这三个都是以官服为蓝本开发的，插件能互相兼容。只是优化之间存在问题，无需多疑。

倘若服务端下载实在过慢，读者可以考虑从 MCBBS 等论坛上或是从 樱花镜像（Sakura Mirror） 这类个人镜像站上获取服务端文件，这里不再赘述

O1 服务端试运行

服务端文件并不大，考虑到国外下载较慢的问题，可能需要等待一些时间。服务端程序都是一个 .jar 后缀结尾的文件。什么？你看不到后缀？

请在计算机资源管理器->文件->界面把你的.jar后缀找回来

如果你是Windows系统，谨记.jar的应用程序不可以直接打开，运行服务端文件的方法是使用脚本。将你的服务端文件置于一个空文件夹（不在C盘，不要中文名，空的就行，别放桌面了），并在此文件夹下新建一个文本文档（.txt），用自带的记事本向里面写入以下内容：

@echo off
title Console
java -Xms1g -Xmx1g -jar [在这里输入自己的服务端文件名称（不带后缀）].jar
pause

这个脚本中的参数可以根据自己的电脑配置进行修改，模板如下：

@echo off
title [窗口标题]
java -Xms[服务端程序启动时分配的内存大小] -Xmx[服务端程序最大占用系统内存资源上限] -jar [在这里输入自己的服务端文件名称（不带后缀）].jar
pause

假设我们的服务器设定为启动服务器时占用内存500MB，但运行功率开到最大时，消耗的内存也不能超过1GB（1024MB），可以将Xms与Xmx参数修改为：

-Xms500m -Xmx1g

将更改的参数加入到源代码中，总的脚本修改后的样子：

@echo off
title Hello World Server
java -Xms500m -Xmx1g -jar PaperSpigot_1.16.5_latest.jar
pause

（不建议将启动内存设置到你设备总内存的1/3以上，修改参数后保存。并将后缀名从.txt改成.bat（无视Windows的提示），然后你就得到了一个命令行文件，Windows默认图标是齿轮与窗口的组合。至此Windows服务端控制台系统就搭建好了，服务器可以通过双击该文件打开命令行的方式启动。

如果你是Linux系统，那么你有下面几种方法：

  • 直接使用java -jar命令启动服务器（同上）
  • 使用脚本启动

当然上面两种方式只是形式上的不同，Linux毕竟是命令行为主的系统。使用命令行快一些。如果你在未来保证不了自己能记得住命令，可以使用bash shell命令行文件。在服务端同文件夹下建立Start.sh，并用vim编辑器打开该服务端。输入下面的命令并保存。

#!/bin/bash[
java -Xms[起始内存] -Xmx[最大内存] -jar [在这里输入自己的服务端文件名称（不带后缀）].jar

参数原理和Windows一样，将Xms和Xmx都调成自己计算机匹配的数值即可。记得保存后使用bash Start.sh命令。

System Info: Java 1.8 (Java HotSpot(TM) 64-Bit Server VM 25.211-b12) Host:  Windows 00.00 (amd64)
Loading libraries, please wait...
[xx:xx:xx INFO]: You need to agree to the EULA in order to run the server. Go to eula.txt for more info.
请按任意键继续. . .

服务器在第一次运行时会出现一个常见的问题，在安装过程中会突然停止，并显示上列信息。

原因在于Mojang的eula协议，这个协议默认是关闭的，你可以在服务端目录下的eula.txt内找到：

eula=false

改为：

eula=true

重新运行你的服务器，服务器在运行过程中会释放一部分文件。留意控制台的内容，一般在弹出下面这条信息后服务器就进入正常工作状态了：

**略去上列内容
[10:15:17 INFO]: Preparing spawn area: 100%
[10:15:17 INFO]: Preparing spawn area: 100%
[10:15:18 INFO]: Preparing spawn area: 100%
[10:15:18 INFO]: Preparing spawn area: 100%
[10:15:18 INFO]: Loaded 441 spawn chunks for world world
[10:15:18 INFO]: Time elapsed: 10537 ms
[10:15:18 INFO]: Done (22.524s)! For help, type "help" <-这条内容代表完成加载

至此，原服务端搭建完毕。

原服务端的文件结构

服务器的原理就是个常开的网络应用程序，如果你想让服务器长期运行，你就要将服务端长期地挂在电脑上。你也可以租一台网络主机（虽然这对要求不是很高的玩家来说确实有点得不偿失），将其挂在服务器上。由于是长期开启，所以你的电脑的硬件和网络资源也相当于被长期占用了一部分，用低端机子开服仍然有待商榷。

如果你想进入这个服务器，你可以将已经打开的服务器挂在一旁，在游戏内的多人游戏中添加服务器，地址栏输入localhost:25565，单击这个服务器进入游戏。这个地址在局域网的各个地方生效。如果你想让外网的玩家游玩到你的服务器，可以参考后期章节部分，我们会讲到如何通过映射内网地址向外网开放你的服务器。

一些基本操作需要等到下章讲到，但是作为常识，你需要先记住：

如果你要停止服务器运行，切记不可单击关闭按钮，在窗口中输入stop即可停止服务器。

如果你想方便快捷的重启服务器，可以输入reload（如果有提示，请输入reload confirm确认即可）即可。

如果你想更换世界，请在服务器停止状态下，将world文件夹的内容更换为你的世界的内容，并启动服务器即可。

如果你是非正版（盗版）用户，进入服务器会出现无法过验证的问题导致无法进入服务器，你可以打开server.properties中，找到online-mode=true一项，并将其true改为false，非正版玩家就能进入服务器了。
